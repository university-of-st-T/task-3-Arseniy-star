# task_3

### Задача «Алхимическая лаборатория»

#### Описание
Вы разрабатываете систему управления инвентарём для алхимической лаборатории. В вашем распоряжении есть словарь `inventory`, который хранит текущее количество доступных ингредиентов (ключ – название ингредиента, значение – целое число). Также имеется книга рецептов `recipes`, где каждому зелью сопоставлен словарь требуемых ингредиентов и их количества.

Необходимо реализовать две функции:

1. **`add_ingredient(inventory, ingredient, amount)`** – добавляет указанное количество ингредиента в инвентарь. Если ингредиент уже существует, его количество увеличивается; если нет – создаётся новая запись. Функция ничего не возвращает, а только модифицирует переданный словарь.

2. **`brew_potion(inventory, recipes, potion_name)`** – пытается приготовить зелье с заданным названием.  
   - Если рецепт отсутствует в книге, функция должна сразу вернуть `False`.  
   - Если для всех ингредиентов из рецепта в инвентаре достаточно количества, функция **вычитает** использованные ингредиенты из инвентаря (при этом ингредиенты, количество которых стало равно нулю, желательно удалять из словаря) и возвращает `True`.  
   - Если хотя бы одного ингредиента не хватает (или его нет в инвентаре), инвентарь **не изменяется**, а функция возвращает `False`.

#### Входные данные (гарантируется)
- `inventory` – словарь вида `{ingredient_name: int}` (количество всегда неотрицательное).
- `recipes` – словарь вида `{potion_name: {ingredient_name: needed_amount, ...}}`.
- Все строки (названия ингредиентов и зелий) не пустые и уникальны в пределах своих словарей.
- Количество ингредиентов – целые числа ≥ 1 в рецептах, в инвентаре могут быть нули.

#### Требования к реализации
- Функции должны быть чистыми в смысле изменений: `add_ingredient` всегда изменяет `inventory`, `brew_potion` изменяет его только при успехе.
- При успешной варке все использованные ингредиенты уменьшаются ровно на требуемое количество. Если количество какого-либо ингредиента становится равным нулю, запись о нём **должна быть удалена** из словаря (это упростит дальнейшие проверки).
- Если в рецепте указан ингредиент, которого нет в инвентаре (или он есть, но в недостаточном количестве), варка невозможна.
- Предусмотрите случай, когда `potion_name` отсутствует в `recipes` – возвращаем `False`.

#### Пример использования

```python
# Исходные данные
inventory = {
    "herb": 3,
    "water": 5,
    "mushroom": 2,
    "crystal": 0
}

recipes = {
    "health_potion": {"herb": 2, "water": 1},
    "mana_potion": {"crystal": 1, "water": 2},
    "invisibility_potion": {"mushroom": 3, "water": 2, "herb": 1}
}

# Добавляем ингредиенты
add_ingredient(inventory, "herb", 2)
add_ingredient(inventory, "crystal", 1)
add_ingredient(inventory, "dragon_blood", 4)

print(inventory)  
# {'herb': 5, 'water': 5, 'mushroom': 2, 'crystal': 1, 'dragon_blood': 4}

# Пытаемся сварить несуществующее зелье
print(brew_potion(inventory, recipes, "love_potion"))  # False
# Инвентарь не изменился

# Пытаемся сварить зелье, на которое не хватает грибов
print(brew_potion(inventory, recipes, "invisibility_potion"))  # False

# Добавляем гриб
add_ingredient(inventory, "mushroom", 1)  # теперь mushroom = 3

# Варим invisibility_potion (требует 3 гриба, 2 воды, 1 траву)
print(brew_potion(inventory, recipes, "invisibility_potion"))  # True
print(inventory)  
# Ожидаем: {'herb': 4, 'water': 3, 'crystal': 1, 'dragon_blood': 4} (mushroom удалён)

# Варим mana_potion (требует 1 кристалл, 2 воды)
print(brew_potion(inventory, recipes, "mana_potion"))  # True
print(inventory)  
# Ожидаем: {'herb': 4, 'water': 1, 'dragon_blood': 4} (crystal удалён)

# Варим health_potion (требует 2 травы, 1 воду)
print(brew_potion(inventory, recipes, "health_potion"))  # True
print(inventory)  
# Ожидаем: {'herb': 2, 'dragon_blood': 4} (water удалена)
```

#### Подсказки
- Для проверки наличия ключа в словаре удобно использовать метод `.get()`.
- При удалении ингредиента с нулевым количеством используйте `del inventory[ingredient]`.
- Не забывайте, что при неудачной попытке варки инвентарь не должен измениться – лучше сначала проверить все ингредиенты, а потом уже вычитать.
